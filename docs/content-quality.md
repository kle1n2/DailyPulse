# DailyPulse 内容质量策略

## 目标

- 输出可读、可追溯的日报内容。
- 在没有稳定翻译服务时优先保证可读性，而不是强行生成伪中文。
- 明确展示栏目缺口和失败来源，不用历史内容冒充当天内容。

## 当前策略

- 标题展示优先使用 `title_display`，默认回退到英文原文。
- 摘要展示优先使用 `summary_display`，没有稳定中文来源时生成保守中文导语。
- 新闻与论文都保留原始来源、发布时间、评分和跳转链接。
- 日报增加 `coverage` 与 `quality` 字段，显式记录：
  - 各栏目完成度
  - 论文供给状态
  - 抓取失败来源
  - 当前生成策略版本

## 已知限制

- RSS 本身质量不稳定，摘要可能只来自 feed snippet，不等同于全文摘要。
- 当前没有接入外部翻译或 LLM 服务，因此不会生成高质量中文改写。
- AI 栏目仍依赖有限来源，虽然已增加补充源，但仍可能填不满。
- 去重仍以 URL 和标题相似度为主，不保证完全消除跨媒体重复。

## 抽检基线

每次改动抓取或排序逻辑后，至少抽检以下内容：

- 3 个新闻栏目，确认标题、摘要、来源、时间可读。
- 2 条论文/博客，确认高亮信息和原文链接正常。
- 任意一份日报，确认 `coverage.overall_status` 与实际栏目数量一致。
- `quality.sourceFailures` 数量与日志表现一致。

## 后续改进方向

- 引入更稳的翻译/摘要能力，再决定是否恢复中文主标题。
- 增加来源健康度统计与连续失败告警。
- 为高重复来源增加跨站点聚类去重。
